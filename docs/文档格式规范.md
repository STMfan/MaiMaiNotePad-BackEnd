# 文档格式规范

> 本文档定义了 MaiMaiNotePad 项目文档的统一格式规范

## 文档结构

### 1. 文档头部

每个文档应包含：
- 标题（一级标题）
- 简介（引用块，一句话描述）

```markdown
# 文档标题

> 文档简介：一句话描述文档内容和用途
```

### 2. 文档主体

文档主体应包含：
- 概述部分
- 主要内容章节（使用二级、三级标题）
- 代码示例（使用代码块）
- 相关链接

### 3. 文档尾部

每个文档必须包含统一的文档信息表格：

#### 标准格式（大多数文档）

```markdown
---

**文档信息**

| 项目 | 内容 |
|------|------|
| 创建日期 | YYYY-MM-DD |
| 最后更新 | YYYY-MM-DD |
| 维护者 | CorrectPath, A-Dawn, cuckoo711 |
| 状态 | [状态标记] |
```

#### 特殊格式

**主要参考文档**（API文档、架构文档、数据模型等）：
```markdown
---

**文档信息**

| 项目 | 内容 |
|------|------|
| 文档版本 | X.X |
| 创建日期 | YYYY-MM-DD |
| 最后更新 | YYYY-MM-DD |
| 维护者 | CorrectPath, A-Dawn, cuckoo711 |
| 状态 | [状态标记] |
```

**更新日志**（持续更新，不标注最后更新）：
```markdown
---

**文档信息**

| 项目 | 内容 |
|------|------|
| 创建日期 | YYYY-MM-DD |
| 维护者 | CorrectPath, A-Dawn, cuckoo711 |
| 状态 | 📝 更新日志 |

> 注：更新日志持续更新，不标注"最后更新"日期
```

**自动生成文档**（错误码文档等）：
- 不添加文档信息表格
- 由生成脚本自动添加时间戳

## 文档状态标记

使用统一的 emoji 标记文档状态：

| 状态 | 标记 | 说明 |
|------|------|------|
| 已完成 | ✅ | 项目已完成，文档记录最终结果 |
| 进行中 | 🚧 | 项目正在进行，文档持续更新 |
| 计划中 | 📋 | 项目尚未开始，文档为计划文档 |
| 参考文档 | 📝 | 参考文档，不涉及具体项目 |
| 更新日志 | 📝 | 版本更新记录 |
| 整理记录 | 📝 | 整理和总结文档 |
| 文档索引 | 📝 | 文档目录和索引 |

## 命名规范

### 文件命名
- 使用中文名称（如：`测试配置指南.md`）
- 避免使用空格，直接连写或使用下划线
- 保持名称简洁明了
- 特殊文件：`.template.md`（模板文件）

### 标题命名
- 一级标题：文档标题
- 二级标题：主要章节
- 三级标题：子章节
- 四级标题：详细说明（谨慎使用）

## 格式规范

### 代码块
使用三个反引号标记代码块，并指定语言：

````markdown
```python
def example():
    pass
```

```bash
./manage.sh test
```
````

### 表格
使用 Markdown 表格格式：

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
```

### 列表
- 无序列表使用 `-`
- 有序列表使用数字
- 嵌套列表使用 2 个空格缩进

### 链接
- 内部链接：`[文档名称](./path/to/doc.md)`
- 外部链接：`[链接文本](https://example.com)`
- 相对路径优先

### 强调
- 粗体：`**文本**`
- 斜体：`*文本*`
- 代码：`` `代码` ``

## 文档分类和格式

### 1. 主要参考文档
包含"文档版本"字段，用于重要的参考文档：
- API文档
- 架构文档
- 数据模型文档

### 2. 一般文档
不包含"文档版本"字段，包含"创建日期"和"最后更新"：
- 配置文档
- 开发指南
- 测试文档
- 迁移说明

### 3. 更新日志
特殊格式，不包含"文档版本"和"最后更新"：
- 只包含"创建日期"
- 持续更新，不标注最后更新时间

### 4. 自动生成文档
不添加文档信息表格：
- 错误码文档（由脚本生成）
- 其他自动生成的文档

## 维护者署名

所有文档的维护者统一为：

```
CorrectPath, A-Dawn, cuckoo711
```

不使用其他形式的署名，如：
- ❌ 开发团队
- ❌ MaiMaiNotePad 开发团队
- ❌ 项目组

## 文档模板

参考 `docs/.template.md` 文件作为新文档的模板。

## 文档更新流程

1. **创建新文档**
   - 复制 `.template.md` 模板
   - 填写文档内容
   - 更新文档信息表格

2. **更新现有文档**
   - 修改文档内容
   - 更新"最后更新"日期
   - 如有必要，更新版本号

3. **更新文档索引**
   - 在 `docs/README.md` 中添加新文档链接
   - 更新相关章节

4. **提交变更**
   - 使用清晰的提交信息
   - 说明文档变更内容

## 文档审查

### 定期审查
- 每月审查一次文档准确性
- 及时更新过时信息
- 删除不再需要的文档
- 合并重复内容

### 审查清单
- [ ] 文档格式符合规范
- [ ] 文档信息表格完整
- [ ] 维护者署名正确
- [ ] 代码示例可运行
- [ ] 链接有效
- [ ] 内容准确无误

## 常见问题

### Q: 如何确定文档状态？
A: 根据文档内容和项目状态选择合适的标记：
- 项目完成 → ✅ 已完成
- 项目进行中 → 🚧 进行中
- 未开始的计划 → 📋 计划中
- 一般参考文档 → 📝 参考文档

### Q: 文档版本号如何管理？
A: 
- 小改动（修正错误、更新日期）：不变更版本号
- 内容更新（新增章节、重要修改）：递增小版本号（1.0 → 1.1）
- 重大改版（结构调整、全面重写）：递增大版本号（1.x → 2.0）

### Q: 如何处理旧文档？
A:
- 如果内容过时但有参考价值：添加"已过时"标记
- 如果完全不再需要：删除文件并从索引中移除
- 如果需要归档：移动到 `docs/archive/` 目录
