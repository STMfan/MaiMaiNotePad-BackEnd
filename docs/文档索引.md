## 📝 文档规范

### 文档格式
- 使用 Markdown 格式
- 使用中文编写（除代码示例外）
- 包含清晰的标题层级
- 添加代码示例和使用说明

### 文档命名
- 使用中文名称（如：`测试文档.md`）
- 避免使用空格，使用下划线或直接连写
- 保持名称简洁明了

### 文档结构
1. **标题**: 清晰的文档标题
2. **简介**: 文档目的和适用范围
3. **内容**: 详细说明和示例
4. **相关链接**: 链接到相关文档

### 更新频率
- **API文档.md**: 每次 API 变更后更新
- **更新日志.md**: 每次版本发布后更新
- **错误码文档.md**: 修改错误消息后自动生成
- **其他文档**: 根据需要更新
