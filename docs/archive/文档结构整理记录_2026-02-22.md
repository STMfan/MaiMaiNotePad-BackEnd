## 后续维护建议

### 1. 定期审查

每季度审查一次文档：
- 检查文档是否过时
- 更新内容和链接
- 归档已完成的任务型文档

### 2. 新文档规范

创建新文档时：
- 使用 `.template.md` 模板
- 遵循文档格式规范
- 更新 `docs/README.md` 索引
- 明确文档类型（指南/参考/任务）

### 3. 归档流程

文档完成使命后：
1. 移至 `docs/archive/`
2. 更新 `docs/archive/README.md`
3. 从主索引中移除
4. 提交变更并说明原因

### 4. 文档分类

建议使用以下分类：
- 📝 指南文档 - 长期有效的使用指南
- 📚 参考文档 - API、架构等参考资料
- ✅ 任务文档 - 完成后应归档
- 📊 分析报告 - 阶段性分析，定期更新或归档
